## Refined Sprint Plan (Sprint 3A): Make Sparks Real

### Objective

Ship a reliable, daily devotional system that pulls real content from the database, is schedulable via Admin, and feels alive through proper empty-states—ready for January 2026 outreach.

### Phase 1 Deliverables (must ship)

#### 1) Schema changes (MVP publishing model)

Add:

* `status` (draft | scheduled | published | archived)
* `publishAt` (datetime)
* `dailyDate` (date)
* `featured` (boolean)
* `mediaUrl` (string) + `mediaMeta` (JSON) as your pragmatic media container

**Why this matters:** it enables scheduling, avoids hardcoded content, and allows “Today’s Devotional” to be deterministic.

#### 2) Home page becomes real

* Replace placeholders in **DailySparks.tsx**
* Pull **featured + latest published** sparks from DB
* Respect status/publishAt rules (don’t show drafts)

#### 3) “Today’s Devotional” (Pattern 1)

* Determine “today” using **Europe/London**
* Fetch devotional where `dailyDate == today` AND `status == published`
* Fallback logic:

  * If none for today: show most recent published devotional
  * If none at all: show empty-state UX

#### 4) Admin scheduling UI (minimal but powerful)

Add:

* Status selector
* Publish date/time picker
* Daily date picker (for devotionals)
* Featured toggle
* Validation rules for publish (e.g., title + mediaUrl + category + thumbnail)

#### 5) Empty-state UX

Define empty-states for:

* No published sparks
* No devotionals scheduled for today
* Filter category returns 0 results

**Why this matters:** empty-states prevent the product feeling “broken” in early rollout, and they guide admins/users to action.

## Phase 2: Content Seeding (run in parallel)

### Objective

Create 15–30 seed sparks that look and behave like real content, so the app feels alive immediately.

### What “good seed content” looks like

Each seed item includes:

* Title, description, category
* mediaType + mediaUrl + thumbnail (vertical 9:16)
* scriptureRef
* duration
* status + publishAt
* devotionals also include dailyDate for at least 2–3 weeks (so you can test rotation properly)

### Recommendation: seed in batches

* **Batch 1 (Week 1):** 7 devotionals scheduled for a full week
* **Batch 2:** 5 testimonies
* **Batch 3:** 5 worship clips
* **Batch 4:** 5 “quick-reads” for high-frequency engagement

This creates both **daily dependability** and **feed variety**.
