# A) Implementation Architecture (Build into reawakened.one)

Because I don’t know your current site stack (WordPress / Webflow / custom), here are three proven ways to integrate without repainting your whole website.

## Option 1 (Recommended): App inside your site at `/vision`

**Best when:** you want the cleanest UX, best performance, and future expansion (community, cohorts, etc.).
**How:** a Next.js/React app served at `reawakened.one/vision` via reverse proxy or same hosting.

* `reawakened.one` marketing pages stay as-is
* `/vision/*` routes go to the Vision Pathway app
* Shared brand header/footer optional

**Pros:** best UX, scalable, fast iteration
**Cons:** requires proper app hosting + auth + database

## Option 2: Subdomain app `app.reawakened.one`

**Best when:** your main site is managed separately (e.g., WordPress/Webflow) and you want zero routing complexity.
**How:** host the app on `app.reawakened.one`, link from your main site.

**Pros:** simplest deployment separation
**Cons:** “feels” slightly separate unless styled identically

## Option 3: Embedded experience (Webflow/WordPress embed)

**Best when:** you need a quick launch.
**How:** embed app screens in an authenticated wrapper (not ideal long-term).

**Pros:** fastest initial integration
**Cons:** weaker performance, can feel less “native,” harder to maintain

### My strong recommendation

Go with **Option 1 or 2** so the experience feels premium and smooth on mobile (Gen Z will notice).

# B) PDF Layout & Export Spec (Print-perfect, “very aesthetic”)

These exports are a major “delight moment.” They should look like the visuals you shared: calm, minimal, elegant, lots of whitespace.

## B1) Export types (4)

1. **Vision Booklet PDF** (all completed sections; the “main” export)
2. **90-Day Plan PDF** (the action roadmap)
3. **Habit Tracker PDF** (monthly grid)
4. **Vision Board PDF** (collage page / words / anchors)

## B2) Page sizes supported

* **A4**: 210 × 297mm
* **US Letter**: 8.5 × 11in

**Rule:** the export service must render both with consistent typography scale and margins.

## B3) Typography (clean + modern + elegant)

Use web-safe, licensed, and available via Google Fonts (easy for HTML→PDF).

**Recommended pairing**

* Headings (serif): `Playfair Display` (or `Cormorant Garamond` if you want softer)
* Body (sans): `Inter` (or `Source Sans 3`)

**Type scale**

* H1: 34–40px (export), tracking slightly tight
* H2: 18–22px
* Body: 11–12px (print-optimized)
* Captions: 9–10px

## B4) Color palette (matches your aesthetic)

* Paper: warm off-white (#F7F4EF or similar)
* Ink: soft charcoal (#2A2A2A)
* Secondary ink: muted gray (#6B6B6B)
* Accent: soft lavender (#7B6CFF-ish, used sparingly)
* Line art: warm gray-beige (#B9B0A6)

**Rule:** accent color appears only on headers, section dividers, small highlights — not everywhere.

## B5) Layout system

* 12-column grid for A4/Letter
* Margins:

  * A4: 18mm top/bottom, 16mm left/right
  * Letter: 0.6in top/bottom, 0.55in left/right
* Section spacing: generous whitespace; never cramped

## B6) Signature visual language (what makes it “Reawakened”)

* Thin-line circles/petals/wheels (0.5–0.75pt line)
* Quiet section dividers
* Minimal icons (optional)
* Strong hierarchy: title → soft subhead → gentle instruction → user content

## B7) HTML-to-PDF template approach (Playwright/Puppeteer)

You’ll render HTML templates with CSS print rules.

Here’s a **starter CSS spec** you can hand to devs (edit palette and fonts as needed):

```css
/* Print tokens */
:root{
  --paper: #F7F4EF;
  --ink: #2A2A2A;
  --muted: #6B6B6B;
  --line: #B9B0A6;
  --accent: #7B6CFF;

  --serif: "Playfair Display", serif;
  --sans: "Inter", system-ui, -apple-system, Segoe UI, Roboto, Arial, sans-serif;

  --h1: 38px;
  --h2: 20px;
  --body: 12px;
  --caption: 10px;

  --radius: 14px;
}

@page {
  size: A4;
  margin: 18mm 16mm;
  background: var(--paper);
}

.page{
  background: var(--paper);
  color: var(--ink);
  font-family: var(--sans);
  font-size: var(--body);
  line-height: 1.45;
}

.h1{
  font-family: var(--serif);
  font-size: var(--h1);
  letter-spacing: -0.5px;
  margin: 0 0 6px 0;
}

.h2{
  font-family: var(--serif);
  font-size: var(--h2);
  margin: 20px 0 8px;
}

.subhead{
  color: var(--muted);
  font-size: 11px;
  margin: 0 0 18px 0;
}

.rule{
  height: 1px;
  background: color-mix(in srgb, var(--line) 65%, transparent);
  margin: 14px 0;
}

.card{
  border: 1px solid color-mix(in srgb, var(--line) 65%, transparent);
  border-radius: var(--radius);
  padding: 14px 14px;
  background: rgba(255,255,255,0.35);
}

.label{
  font-size: 10px;
  letter-spacing: 0.12em;
  text-transform: uppercase;
  color: var(--muted);
  margin-bottom: 6px;
}

.value{
  font-size: 12px;
  color: var(--ink);
  white-space: pre-wrap;
}

.accent{
  color: var(--accent);
}

.footer{
  position: running(footer);
  font-size: var(--caption);
  color: var(--muted);
}

@page{
  @bottom-center{
    content: counter(page);
    font-family: var(--sans);
    font-size: 10px;
    color: var(--muted);
  }
}

/* Letter sizing override */
.letter {
  /* devs can switch @page size at render time */
}
```
